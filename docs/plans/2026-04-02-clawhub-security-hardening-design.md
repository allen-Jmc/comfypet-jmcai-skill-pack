# 2026-04-02 ClawHub 安全加固与告警消除设计

## 背景
当前 `comfypet-jmcai-skill` 在 ClawHub 发布页面被标记为“可疑 (Suspicious)”。主要原因归结为 VirusTotal 的代码洞察检测到了“任意本地文件读取并上传”的模式，以及部分元数据链接可能触发了静态分析器的告警。

## 设计目标
1. **消除“任意读取”告警**：通过白名单机制限制可上传的文件类型。
2. **规范化元数据**：确保所有链接指向官方域，消除自动化扫描的疑虑。
3. **提升信任度**：在文档中显式声明安全实践。

## 方案细节

### 1. 代码层面：安全上传围栏 (Secure Upload Guard)
在 `jmcai_skill.py` 中引入 `ALLOWED_EXTENSIONS` 校验。

- **逻辑控制**：
    - 在 `upload_local_file` 开始前，检查文件后缀。
    - 如果后缀不在允许列表（图片/视频常见格式）中，直接报错并停止上传。
- **允许列表**：
    - 图片：`.jpg`, `.jpeg`, `.png`, `.webp`, `.bmp`
    - 视频：`.mp4`, `.mov`, `.avi`, `.gif`
    - 音频：`.wav`, `.mp3`, `.ogg`, `.flac`, `.m4a`
- **进阶校验**：
    - 使用 `Path(local_path).resolve()` 获取真实路径，防止路径遍历攻击。

### 2. 元数据层面：规范化链接与描述
- **SKILL.md**：
    - 检查 `metadata.openclaw.homepage` 确保使用完整 GitHub URL。
    - 增加 `## Security & Privacy` 章节，解释为什么需要上传文件（ComfyUI 远程渲染需求）。
- **config.example.json**：
    - 保持 `127.0.0.1`，但在描述中强调其本地回环性质。

### 3. 分发层面
- 确保 `release/clawhub-publish.ps1` 和相关文档不包含 IP 缩短或非官方下载链接。

## 验证计划
1. **功能验证**：提交一个正常的图片/视频工作流，确认 `doctor` 和 `run`（含上传）依然正常工作。
2. **安全验证**：尝试手动输入一个敏感文件路径（如 `README.md` 或 `.git/config`）作为图片参数，确认系统会拦截并报错。
3. **静态扫描验证**：本地运行静态检查工具（若有）或重新提交至 ClawHub 观察扫描结果。

## 风险评估
- **低风险**：由于禁止上传的文件本身就不能被 ComfyUI 处理，因此不会对正常工作流产生任何负面影响。
